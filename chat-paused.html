Chat paused — 给角色添加封窗权

让 AI 角色可以主动「关闭」对话窗口。它可以冷静，可以催你去睡觉，也可以制造一点紧张感——你得 Give Plead 才能回来。

先试一下长什么样

点击后从底部滑出，和 Claude 官端一个感觉

思路

核心是给模型加一个 close_window 工具。角色在想关闭对话时调用它，传入一个 reason（关闭原因）。后端检测到这个工具调用，把原因包装进响应 JSON 返回给前端，前端收到后弹出 Chat paused 弹窗、禁用输入框。

后端 · 第一步 定义工具

把 close_window 加进你的工具列表。reason 是必填参数，角色会在这里写关闭原因，直接给用户看。

Anthropic 格式（Claude 系列）

Python 
# 加入你的 TOOLS 列表
{
    "name": "close_window",
    "description": (
        "暂时关闭对话窗口，给用户弹出「Chat paused」通知。"
        "当你需要冷静、或想让用户去做别的事、或想制造紧张感时调用。"
        "调用前先在正文说一句告别，再调用工具。"
        "reason 填关闭原因，一句话，会直接显示给用户。"
    ),
    "input_schema": {
        "type": "object",
        "properties": {
            "reason": {
                "type": "string",
                "description": "关闭原因，一句话，温柔或强硬都行。",
            }
        },
        "required": ["reason"],
    },
},

OpenAI / OpenRouter 格式

Python 
{
    "type": "function",
    "function": {
        "name": "close_window",
        "description": "暂时关闭对话窗口，给用户弹出 Chat paused 通知...",
        "parameters": {
            "type": "object",
            "properties": {
                "reason": {
                    "type": "string",
                    "description": "关闭原因，一句话。",
                }
            },
            "required": ["reason"],
        },
    },
},

后端 · 第二步 检测工具调用

在你处理工具调用结果的地方加上 close_window 的分支。把 reason 以 "close_window:原因" 的格式存进 tools_called 列表，方便后续提取。

Python — Anthropic 收集循环 
for block in content:
    if block.get("type") == "tool_use":
        # ... 你已有的其他工具处理 ...

        elif (block.get("name") == "close_window"
              and not any(t.startswith("close_window:") for t in tools_called)):
            reason = block["input"].get("reason", "")
            tools_called.append(f"close_window:{reason}")
Python — OpenRouter tool_calls 循环 
elif name == "close_window":
    reason = args.get("reason", "")
    if any(t.startswith("close_window:") for t in tools_called):
        result_text = "重复操作忽略。"
    else:
        tools_called.append(f"close_window:{reason}")
        result_text = "窗口已关闭。"

tool_results 里也要给 close_window 一个返回文本，告诉模型「操作已执行」：

Python 
elif tb.get("name") == "close_window":
    result_text = "窗口已关闭。"

后端 · 第三步 返回给前端

在 /api/chat 的 return 之前，把 close_window:... 从 tools_called 里提取出来，单独放进 window_closed 字段，前端 tools_called 里就不会看到它了。

Python — /api/chat 路由末尾 
cw_entry = next(
    (t for t in (tools_called or [])
     if isinstance(t, str) and t.startswith("close_window:")),
    None
)
window_closed = {"reason": cw_entry[len("close_window:"):]} if cw_entry else None
tools_for_frontend = [
    t for t in (tools_called or [])
    if not (isinstance(t, str) and t.startswith("close_window:"))
]

return jsonify({
    "reply": reply,
    "replies": replies,
    # ... 你已有的其他字段 ...
    "tools_called": tools_for_frontend,
    "window_closed": window_closed,  # None 或 {"reason": "..."}
})

② 弹窗完整逻辑（放在初始化代码里）

JavaScript 
const cwModal     = document.getElementById("closeWindowModal");
const cwDescEl    = document.getElementById("cwDesc");
const cwErrorEl   = document.getElementById("cwError");
const cwGivePlead = document.getElementById("cwGivePlead");
const cwNoCont    = document.getElementById("cwNoContinue");
const cwLearnBtn  = document.getElementById("cwLearnMore");

let _cwReasonText = "";
let _cwReasonVisible = false;

function showCloseWindowModal(reason) {
  _cwReasonText    = reason;
  _cwReasonVisible = false;
  cwDescEl.textContent   = "";
  cwErrorEl.textContent  = "";
  cwLearnBtn.textContent = "No Learn More";
  cwGivePlead.disabled   = false;
  cwGivePlead.textContent = "Give Plead";
  inputEl.disabled = true;   // 禁用输入框
  sendBtn.disabled = true;
  cwModal.classList.remove("hidden");
}

function hideCloseWindowModal() {
  cwModal.classList.add("hidden");
  inputEl.disabled = false;
  sendBtn.disabled = false;
}

// No Learn More / No Learn Less 切换
cwLearnBtn.addEventListener("click", () => {
  _cwReasonVisible = !_cwReasonVisible;
  cwDescEl.textContent   = _cwReasonVisible ? _cwReasonText : "";
  cwLearnBtn.textContent = _cwReasonVisible ? "No Learn Less" : "No Learn More";
});

// 不许出去！
cwNoCont.addEventListener("click", () => showToast("不许！"));

// Give Plead — 求角色放行
cwGivePlead.addEventListener("click", async () => {
  cwGivePlead.disabled    = true;
  cwGivePlead.textContent = "传达中……";
  cwErrorEl.textContent   = "";
  try {
    const resp = await fetch("/api/plead", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ character_id: currentChar }),
    });
    if (!resp.ok) throw new Error("HTTP " + resp.status);
    const data = await resp.json();
    hideCloseWindowModal();
    // 把角色的回复渲染成气泡（用你自己的渲染逻辑）
    renderAIReply(data);
  } catch (e) {
    cwErrorEl.textContent   = "没送到，再试一次？";
    cwGivePlead.disabled    = false;
    cwGivePlead.textContent = "Give Plead";
  }
});

后端 · 可选 /api/plead 端点

Give Plead 和「和好按钮」是两种完全不同的语境，建议分开。/api/plead 的系统提示是「用户在窗口外求你放她进来」，而不是「哄哄我」。

Python — Flask 
@app.route("/api/plead", methods=["POST"])
def plead():
    body = request.json or {}
    character_id = body.get("character_id", "default")
    if character_id not in CHARACTERS:
        return jsonify({"error": "未知角色"}), 400
    char = CHARACTERS[character_id]
    plead_msg = (
        "[系统提示：你刚才关闭了对话窗口，"
        "用户在窗口外面求你了，说「求求你放我进来嘛」"
        "——你要怎么回应？用你的风格，不要提及这是系统触发的]"
    )
    reply, *_ = ask_character(char, "default", plead_msg)
    if not reply or not reply.strip():
        reply = "(沉默片刻)"
    reply_id = save_message("default", character_id, "model", reply)
    replies = reply.split("||") if "||" in reply else reply.split("\n\n")
    replies = [r.strip() for r in replies if r.strip()]
    return jsonify({"reply": reply, "replies": replies, "reply_id": reply_id})
